The Applied GenAI Stack

A structured 3-pass learning path from architecture thinking → working prototypes → production-depth knowledge.

PASS 1

Architecture Thinking

Understand how modern GenAI systems are structured and why each piece exists.

Core Pipeline
Document Ingestion → Chunking → Embedding → Vector Store → Retrieval → Context Augmentation → Generation
Basic
Naive RAG Conversational RAG Hybrid RAG (sparse + dense) Multi-hop RAG
Intermediate
Query Rewrite RAG HyDE (Hypothetical Document Embedding) Multi-source RAG Graph RAG
Advanced
Corrective RAG (CRAG) Self-RAG Adaptive RAG Agentic RAG
Advanced architectures add validation loops, multiple retrieval steps, or autonomous agent control to improve reasoning and reduce hallucinations.
Key Concepts
Embeddings Vector Similarity Approximate Nearest Neighbor (ANN) Index Structures Metadata Filtering
Important Vector DBs
Pinecone Weaviate Qdrant Chroma FAISS Milvus pgvector
Key tradeoff to understand: Managed (Pinecone) vs self-hosted (Qdrant, Milvus). For learning, start with Chroma (simple, local). For production, understand HNSW vs IVF indexing.
Core Formula
LLM (brain) + Tools (hands) + Planning (strategy) + Memory (context) = Agent
Basic

Intermediate

Memory Types
Short-term Memory (conversation buffer) Long-term Memory (persistent store) Episodic Memory (past events) Semantic Memory (facts & knowledge) Vector Memory Graph Memory
Key insight: Memory retrieval is itself a RAG problem. The agent embeds past interactions, stores them, and retrieves relevant memories at query time. This is where RAG and agents deeply converge.
Concepts

Common Tools
APIs Databases Search Engines Code Execution Calculators Browser / Computer Use
Frameworks to Know
LangChain LangGraph LlamaIndex CrewAI Haystack Semantic Kernel
Recommendation: Start with LangChain for RAG, LangGraph for agents, LlamaIndex for data-heavy RAG. Use CrewAI when exploring multi-agent patterns.
PASS 2

High-Level Building

Build real working systems. Don't optimize - just make it work end-to-end.

01 Simple RAG

Load a PDF → chunk it → embed with OpenAI/Cohere → store in Chroma → build a chatbot that answers questions from the document.

PDF parsing Chunking Embeddings Vector DB Prompt template

02 Conversational RAG

Extend project 1. Add chat memory so follow-up questions work. Handle "what did I ask earlier?" and contextual references.

Chat history Session memory Context window management

03 Multi-Source RAG

Query across multiple sources: a vector DB, a SQL database, a web search API. Route the query to the right source.

Query routing SQL + Vector hybrid Source selection

04 Agent with Tools

Build an LLM agent that can: search the web, run Python code, call APIs, and retrieve from your vector store. Watch it reason through multi-step tasks.

ReAct loop Tool binding Function calling

05 Agentic RAG

The agent decides when to retrieve, which tool to use, whether the retrieved context is good enough, and when to re-query. This is where RAG and agents merge.

Retrieval decisions Self-critique Adaptive retrieval

06 Multi-Agent System

Build a research assistant with: a planner agent, a retriever agent, a summarizer agent, and a verifier agent. Watch them collaborate.

Agent delegation Message passing Supervisor pattern

Trending

Latest Topics

2024+

Graph RAG

Build knowledge graphs from documents, retrieve over relationships instead of flat vector similarity. Better for complex reasoning.

2024+

Agentic RAG

Agents that autonomously decide when to retrieve, what source to query, and whether retrieved context is good enough.

2025+

Multi-Modal RAG

RAG over images, audio, video — not just text. ColPali for vision-based document understanding without text extraction.

2025+

Context Engineering

The emerging discipline: memory + retrieval + tool context + prompt structure as one unified design problem.

2025+

MCP (Model Context Protocol)

Anthropic's open standard for connecting agents to tools and data sources. Becoming the USB-C of agent tool use.

2025+

LLM Operating Systems

Agent runtimes, tool ecosystems, AI workflows. LLMs as the kernel of a new kind of software platform.
